Kubernetes Error Reference

Error Category Cause Fix
Container Crash Loop Pod The container starts, crashes, is restarted by Kubernetes, crashes again, and so on. Kubernetes applies exponential backoff between restarts (10s, 20s, 40s... up to 5 minutes). Check logs from the last crash: `kubectl logs --previous`. Describe the pod: `kubectl describe pod `. Common causes: application startup failure, missing env vars, missing config, bad entrypoint command.
Image Pull Back Off Pod Kubernetes cannot pull the container image. Kubernetes retries with exponential backoff after repeated failures. Verify the image name and tag. Check registry credentials: `kubectl get secret `. Create an imagePullSecret if using a private registry. Check registry availability from the node.
Image Pull Error Pod Initial failure to pull the image before backoff kicks in. Image name typo, tag not found, or auth failure. Run `kubectl describe pod ` for the exact error. Test the pull manually on the node. Verify registry credentials and imagePullPolicy.
Out of Memory Killed Pod The container exceeded its memory limit and was killed by the Linux OOM killer. The container's `resources.limits.memory` was hit. Increase memory limits in the pod spec. Profile the application for memory leaks. Set realistic `requests` and `limits`. Check `kubectl describe pod ` for `OOMKilled: true` and `Reason: OOMKilled`.
Pod Evicted Pod The node ran out of resources (disk, memory, PID) and Kubernetes evicted pods to reclaim resources. Pods without requests/limits or those below the QoS threshold are evicted first. Set `resources.requests` and `resources.limits` on all containers. Increase node capacity. Check node disk pressure: `kubectl describe node `. Clean up old logs and unused images on nodes.
Pod Stuck in Pending Pod Kubernetes cannot schedule the pod. Common causes: insufficient resources on any node, node selectors or affinity rules cannot be satisfied, taints without tolerations, PVC not bound. Run `kubectl describe pod ` and look at Events. Check node capacity: `kubectl describe nodes`. Verify node selectors, affinity, and taints match available nodes.
Container Config Error Pod Kubernetes cannot create the container due to a misconfiguration. Usually a missing Secret, ConfigMap, or invalid volume mount. Run `kubectl describe pod `. Check that referenced Secrets and ConfigMaps exist in the same namespace: `kubectl get secret,configmap -n `.
Run Container Error Pod Container started but failed immediately, often due to a bad command, missing binary, or wrong working directory. Check `kubectl describe pod ` for the specific error. Verify the container command and entrypoint in the pod spec against what the image provides.
Invalid Image Name Pod The image name in the pod spec is malformed (e.g. contains uppercase, invalid characters, or a bad registry URL format). Correct the image name in the deployment/pod spec. Image names must be lowercase. Validate: `docker pull ` locally.
PostStart Lifecycle Hook Error Pod The postStart lifecycle hook command failed. Kubernetes blocks the container from reaching Running state until the hook completes. Debug the hook command in `kubectl describe pod`. Run the command manually in a test container. Ensure the hook is idempotent and handles race conditions with container startup.
PreStop Hook Timeout Pod The preStop hook ran for longer than `terminationGracePeriodSeconds` and was killed before completing. Increase `terminationGracePeriodSeconds`. Optimize the preStop hook to complete quickly. Ensure graceful shutdown is handled in the application itself, using preStop only for coordination.
Node Not Ready Node The node is unhealthy and not accepting workloads. Causes include kubelet crash, network partition, disk pressure, memory pressure, or PID pressure. SSH to the node and check kubelet: `systemctl status kubelet`. Check node conditions: `kubectl describe node `. Drain and cordon the node if needed: `kubectl drain --ignore-daemonsets`.
Node Disk Pressure Node Available disk space on the node is below the threshold (default: 10% or 100Mi free). Kubernetes will start evicting pods. SSH to the node and free disk space. Remove unused container images: `crictl rmi --prune`. Clean up old logs. Increase node disk size. Set `evictionHard` thresholds in kubelet config.
Node Memory Pressure Node Available memory on the node is below the eviction threshold. Kubernetes will start evicting BestEffort and Burstable pods. Check memory usage on the node. Review pod resource requests and limits. Move pods with high memory usage to larger nodes. Add more nodes to the cluster.
Node PID Pressure Node The number of processes on the node is approaching the system PID limit. Can be caused by fork bombs, runaway processes, or too many containers. Identify runaway processes: `ps aux --sort=-%cpu | head`. Set PID limits per pod: `resources.limits.ephemeral-storage`. Increase system PID limits: `kernel.pid_max` sysctl.
Node Network Not Ready Node The CNI (Container Network Interface) plugin is not running or failed. Pods cannot get IP addresses. Check CNI pod logs: `kubectl logs -n kube-system `. Restart the CNI daemonset. Verify CNI binary is installed on the node at `/opt/cni/bin/`.
Deployment Progress Deadline Exceeded Deployment The deployment did not make progress within `progressDeadlineSeconds` (default: 600s). New pods are not becoming Ready. Check pod status: `kubectl get pods`. Describe the deployment: `kubectl describe deployment `. Look for failed pod events. Check image, resources, and readiness probes.
ReplicaSet Failed to Create Pods Deployment The ReplicaSet controller cannot create pods. Common causes: resource quota exceeded, RBAC missing, invalid pod spec. Check ReplicaSet events: `kubectl describe rs `. Check namespace resource quotas: `kubectl describe resourcequota -n `.
HPA Cannot Scale Scaling HPA cannot scale because: metrics server is not installed, the deployment is missing resource requests (required for CPU-based scaling), or max replicas is already reached. Install metrics-server. Set `resources.requests.cpu` on the container. Check HPA status: `kubectl describe hpa `. Increase `maxReplicas` if needed.
Service Has No Endpoints Service The Service selector does not match any running pod labels, or all matched pods are not Ready. Check endpoints: `kubectl get endpoints `. Verify pod labels match the service selector: `kubectl get pods --show-labels`. Check pod readiness probes.
LoadBalancer External IP Pending Service A Service of type LoadBalancer has been created but no external IP is assigned. The cloud provider controller has not provisioned a load balancer. Verify the cloud controller manager is running. Check service events: `kubectl describe service `. On local clusters (kind, minikube) use NodePort or a tool like MetalLB.
Ingress Backend Service Not Found Networking The Ingress resource references a backend Service that does not exist or has the wrong port. Verify the service name and port in the Ingress spec match an existing Service: `kubectl get svc -n `. Check ingress controller logs.
CoreDNS Resolution Failure Networking Pods cannot resolve DNS names. CoreDNS may be crashed, misconfigured, or a network policy is blocking DNS traffic. Check CoreDNS pods: `kubectl get pods -n kube-system -l k8s-app=kube-dns`. Test DNS from a pod: `kubectl exec -- nslookup kubernetes.default`. Check NetworkPolicy for port 53 blocking.
NetworkPolicy Blocking Traffic Networking A NetworkPolicy is blocking traffic between pods or from pods to external services. Once any NetworkPolicy selects a pod, all traffic not explicitly allowed is denied. Review NetworkPolicy rules: `kubectl get networkpolicy -n `. Test with a temporary allow-all policy. Use `kubectl describe networkpolicy` to see selectors and rules.
PersistentVolumeClaim Stuck Pending Storage No PersistentVolume matches the PVC (access mode, storage class, size). Or the StorageClass provisioner is not running. Check PVC status: `kubectl describe pvc `. List available PVs: `kubectl get pv`. Verify StorageClass exists and provisioner is running: `kubectl get storageclass`.
PersistentVolumeClaim Lost Storage The underlying PersistentVolume was deleted or became unavailable while the claim was still in use. Often caused by manually deleting PVs. This may result in data loss. Restore the volume from backup if available. Never manually delete PVs that are bound to PVCs. Use `reclaimPolicy: Retain` on StorageClasses for critical data.
Secret Not Found Configuration A pod, deployment, or ingress references a Secret that does not exist in the same namespace. Create the missing secret: `kubectl create secret generic --from-literal=key=value -n `. Verify namespace: `kubectl get secrets -n `.
ConfigMap Not Found Configuration A pod spec references a ConfigMap that does not exist in the same namespace. Create the ConfigMap: `kubectl create configmap --from-file=config.yaml -n `. Use `kubectl get configmap -n ` to verify.
RBAC Permission Denied RBAC A pod or user is attempting an API action they do not have permission for. The ServiceAccount lacks the required Role or ClusterRole binding. Check RBAC: `kubectl auth can-i --as=system:serviceaccount::`. Create a Role and RoleBinding granting the required permissions.
ServiceAccount Not Found RBAC The pod spec references a ServiceAccount that does not exist in the namespace. Create the ServiceAccount: `kubectl create serviceaccount -n `. Or set `serviceAccountName: default` to use the default account.
Pod Security Admission Violation Security The pod spec violates the Pod Security Standards policy enforced on the namespace (e.g. requesting privileged mode, hostNetwork, or running as root in a restricted namespace). Check namespace labels: `kubectl get namespace --show-labels`. Adjust pod spec to comply with the policy (remove privileged, set runAsNonRoot, set seccompProfile). Or relax the namespace policy.
Readiness Probe Failing Pod The readiness probe is failing, so the pod is excluded from Service endpoints. The container is running but not passing health checks. Check probe configuration in `kubectl describe pod`. Test the probe endpoint manually from inside the pod: `kubectl exec -- curl localhost:/health`. Increase `initialDelaySeconds` if the app takes time to start.
Liveness Probe Failing Pod The liveness probe is failing, causing Kubernetes to restart the container. The application may have deadlocked or stopped responding. Inspect pod events: `kubectl describe pod`. Check if the probe threshold is too aggressive. Test the probe endpoint: `kubectl exec -- curl localhost:/live`. Increase `failureThreshold` or `timeoutSeconds`.
Startup Probe Failing Pod The startup probe did not succeed within `failureThreshold * periodSeconds`. Kubernetes kills the container before it finishes initializing. Increase `failureThreshold` or `periodSeconds` on the startup probe to give the app more time. Check application startup logs: `kubectl logs `.
Resource Quota Exceeded Configuration Creating or updating resources in the namespace would exceed the configured ResourceQuota (CPU, memory, pod count, service count, etc.). Check quota usage: `kubectl describe resourcequota -n `. Delete unused resources or increase quota. Ensure all pods have resource requests set (required for quota enforcement).
LimitRange Violation Configuration A pod or container is requesting resources outside the range defined by a LimitRange in the namespace. Check LimitRange: `kubectl describe limitrange -n `. Adjust container resource requests and limits to fall within the defined min/max values.
StatefulSet Rolling Update Stuck StatefulSet A StatefulSet rolling update is stuck because a pod is not becoming Ready (often due to PVC issues, readiness probe failures, or anti-affinity blocking scheduling). Check the stuck pod: `kubectl describe pod -`. StatefulSets update in reverse ordinal order and will not proceed past a stuck pod. Fix the underlying pod issue.
Job Backoff Limit Exceeded Job All Job pods failed and the retry limit (`backoffLimit`) has been reached. Kubernetes stops retrying. Check failed pod logs: `kubectl logs `. Fix the underlying issue and recreate the Job. Increase `backoffLimit` if transient failures are expected.
CronJob Missed Schedule CronJob The CronJob missed its scheduled execution window. This can happen if the controller was down or if a previous job run took longer than the scheduled interval. Check `startingDeadlineSeconds` on the CronJob. Set `concurrencyPolicy: Forbid` to prevent overlapping runs. Verify the CronJob schedule with a cron expression validator.
Ingress TLS Certificate Secret Not Found Networking The Ingress TLS section references a Secret that does not exist or has not yet been provisioned by cert-manager. Check cert-manager Certificate resources: `kubectl get certificate -n `. Describe the Certificate to see provisioning errors. Manually create the Secret if cert-manager is not being used.
etcd Quorum Lost Control Plane More than half of etcd members are unavailable, causing the cluster to become read-only. The API server cannot write state. This is a critical control plane failure. Restore etcd from a snapshot. For a 3-node cluster, losing 2 nodes loses quorum. Always run etcd with an odd number of members (3 or 5).
API Server Unreachable Control Plane The Kubernetes API server is down or unreachable. kubectl commands fail with connection refused or timeout errors. Check API server pod: `kubectl get pods -n kube-system` (if possible). SSH to control plane node: `docker ps | grep apiserver` or `crictl ps | grep apiserver`. Check logs and restart if needed.
kube-scheduler Unhealthy Control Plane The scheduler pod is crashed or leader election failed. New pods will stay in Pending state indefinitely. Check scheduler pod: `kubectl get pods -n kube-system -l component=kube-scheduler`. Restart the scheduler pod. Check leader election: `kubectl get lease -n kube-system`.
kube-controller-manager Unhealthy Control Plane The controller manager is not running. Deployments, ReplicaSets, DaemonSets, and Jobs will not reconcile. Check controller-manager pod: `kubectl get pods -n kube-system -l component=kube-controller-manager`. Restart the pod. Check logs for certificate or RBAC issues.

Frequently Asked Questions

What causes Kubernetes CrashLoopBackOff?

A container keeps crashing and Kubernetes restarts it with exponential backoff. Common causes: application crash on startup (check kubectl logs POD --previous), misconfigured env vars or secrets, failed liveness probe, or missing dependencies. Check kubectl describe pod POD_NAME for events.

How do I fix Kubernetes ImagePullBackOff?

Causes: (1) incorrect image name or tag, (2) private registry credentials missing - create an imagePullSecret, (3) Docker Hub rate limiting - authenticate with a Docker Hub account, (4) image does not exist - verify with docker pull locally first.

What does OOMKilled mean in Kubernetes?

The container exceeded its memory limit and the OOM killer terminated it. Fix: check actual usage with kubectl top pod, increase memory limit in resources.limits.memory, investigate memory leaks, and always set both requests and limits in production.
